Section 2: Manual Implementation Tasks

Action Required
The following tasks require manual implementation by you to fully integrate the component.
  • Integration with RightSidebar "Configure" button
    Add a Configure button to the properties panel that opens TypeConfigModal:
    @if (HasTypeConfig(SelectedField.FieldType)) { <button class="btn btn-sm btn-outline" @onclick="OpenTypeConfig"> <i class="bi bi-sliders"></i> @(SelectedField.TypeConfig != null ? "Edit Configuration" : "Configure") </button> } <TypeConfigModal @bind-IsOpen="showTypeConfig" FieldType="@SelectedField.FieldType" ExistingConfig="@SelectedField.TypeConfig" OnSave="HandleTypeConfigSave" />
  • HasTypeConfig helper function
    Add this helper to determine which field types support TypeConfig:
    private bool HasTypeConfig(string fieldType) => fieldType is "DatePicker" or "TimePicker" or "DateTimePicker" or "FileUpload" or "AutoComplete" or "DataGrid";
  • HandleTypeConfigSave implementation
    Handle saving the configuration back to the field:
    private void HandleTypeConfigSave(FieldTypeConfig config) { // Update the field's TypeConfig property SelectedField = SelectedField with { TypeConfig = config }; // Trigger state update }
  • CSS import (if needed)
    The TypeConfigModal.razor.css is a scoped stylesheet that should auto-apply. If styles don't appear, ensure Blazor CSS isolation is working properly.

Component Usage Example
