Section 1: Manual Testing Steps

Start Node

  • Start node displays with green border (#10B981)
    Verify border color matches the design spec
  • Header background is light green (rgba(16, 185, 129, 0.1))
    Check header section has correct tinted background
  • Play icon (bi-play-fill) displays in icon container
    Icon should be white on green background
  • Only bottom handle is visible (no top handle)
    Start nodes connect outward only
  • Trigger type displays correctly in footer (e.g., "Trigger: On Submit")
    Test all trigger types: OnSubmit, OnSave, OnLoad, Manual, Scheduled

End Node

  • End node displays with gray border (#6B7280)
    Verify border color matches the design spec
  • Header background is light gray
    Check header section has correct tinted background
  • Stop icon (bi-stop-fill) displays correctly
    Icon should be white on gray background
  • Only top handle is visible (no bottom handle)
    End nodes receive connections only
  • Completion action displays in footer (e.g., "Action: Submit")
    Test all actions: Submit, Save, Redirect, ShowMessage, CallApi

Step Node

  • Step node displays with indigo border (#6366F1)
    Verify primary color is used for step nodes
  • Top and bottom handles are visible
    Steps can connect in both directions
  • Field count displays correctly (e.g., "8 fields")
    Should show Module.Fields.Length value
  • Required indicator shows when IsRequired=true
    Should display "Required" text in red
  • Skip condition indicator shows when SkipCondition is set
    Yellow badge with "Has skip condition" text

Decision Node

  • Decision node displays with orange border (#F59E0B)
    Verify warning color is used for decisions
  • All 4 handles visible (top, bottom, left, right)
    Decision nodes support multi-directional branching
  • Condition summary displays (e.g., "If fieldId = value")
    Should parse first condition from ConditionGroupModel
  • Yes/No branch badges display correctly
    Yes in green, No in red with icons
  • Custom branch labels work (YesBranchLabel, NoBranchLabel)
    Should override default "Yes"/"No" text

Action Node

  • Action node displays with purple border (#8B5CF6)
    Verify purple color is used for actions
  • Correct icon shows per action type
    SendEmail=envelope, CallApi=globe, SetFieldValue=pencil, etc.
  • Action description displays correctly
    E.g., "Send email to admin@example.com"
  • Email recipients show for SendEmail actions
    Should display recipient list with envelope icon
  • Action type label shows in footer (e.g., "Email Action")
    Test all action types

Common Node Behavior

  • Node selection highlights with colored ring
    Selected state uses node's color for highlight
  • Menu button appears on node hover
    Three-dots vertical icon should show
  • Handles scale 1.3x on hover
    Connection handles should grow on mouseover
  • Handles show crosshair cursor
    Indicates connection mode
  • Node shows move cursor
    cursor: move on node body
  • OnSelected event fires on click
    Parent should receive node ID
  • Dark mode styles apply correctly
    Background and border colors adapt to theme

Usage Example

Example of how to use the new node components in a parent canvas:

@* In WorkflowCanvas.razor *@ @using VisualEditorOpus.Components.Workflow.Nodes @foreach (var node in Nodes) { @switch (node.Type) { case WorkflowNodeType.Start: <WfNodeStart Id="@node.Id" Node="@node" Position="@node.Position" IsSelected="@(SelectedNodeId == node.Id)" OnSelected="HandleNodeSelected" OnConnectionStart="HandleConnectionStart" /> break; case WorkflowNodeType.End: <WfNodeEnd Id="@node.Id" Node="@node" Position="@node.Position" IsSelected="@(SelectedNodeId == node.Id)" OnSelected="HandleNodeSelected" OnConnectionStart="HandleConnectionStart" /> break; // ... other node types } }
